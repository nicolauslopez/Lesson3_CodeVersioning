History of... Git.

While developing Linux, which is open source adn everyone is working on it together, they needed a way to track changes and go back to previous versions if needed. So then the concept of Git was created.

Let's talk... Version Control.

Version Control is needed so when we make modifications to a project we can track it. Lets say we break a program after development and you've invested hours of coding into it, you won't be able to rememeber what you changed and what was there before. Having version control you will be able to see what changes you've made and you can go back to a version that works.

Git. How it works.

Instead of re-uploading all the files Git looks at the changes you've made and if it sees that you updated a file then they will only upload that file and link that one to the rest in the folder. So instead of just making a different version file, Git still has all the files from previous it just updates what needs to be updated.
